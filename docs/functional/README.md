# Functional Docs

Plain-English, user-facing documentation of what the system does and how it is meant to be used. Distinct from `docs/reality/`: reality describes the code as it is today, functional docs describe the product as it should be understood by someone using or reasoning about it. One file per feature area, written without jargon, with worked examples where helpful.

Created as part of the Phase 0 audit-workflow scaffolding. Files here will be produced in **Phase 2 (plain-English functional docs)** of the audit workflow.
