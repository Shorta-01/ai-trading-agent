## Purpose

Single-file chronological index of every decision made during the multi-phase audit and execution workflow. Each detailed decision lives in its own file at `docs/decisions/NNNN-<slug>.md`; this log is the at-a-glance summary so a reader can see the full decision history without opening every file.

## Conventions

- Append-only. Never edit or delete past entries — supersede them with a new entry that references the old one.
- One row per decision, newest at the bottom.
- **ID:** matches the filename of the detailed record (`0001`, `0002`, …).
- **Status:** one of `proposed`, `accepted`, `superseded`, `deprecated`.
- **Phase:** which audit phase produced the decision (`P1`, `P2`, `P3`, `P4`, `P5`).
- **Supersedes / Superseded by:** decision IDs, where applicable.
