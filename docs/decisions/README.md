# Decisions

Lightweight decision records produced during the audit and execution workflow. Each file captures a single decision: context, options considered, decision, and consequences. Numbered sequentially (`0001-<slug>.md`, `0002-<slug>.md`, …).

**Note on overlap with `docs/adr/`:** an existing ADR directory (`docs/adr/`) already holds three pre-audit architecture decisions (database/migrations, IBKR source-of-truth, IBKR integration approach). `docs/decisions/` is reserved for decisions originating from the audit workflow itself — primarily Phase 4 brainstorming outcomes. A future consolidation may merge the two; for now they coexist.

Created as part of the Phase 0 audit-workflow scaffolding. Files here will be produced in **Phase 4 (topic brainstorming → tasks → PRs → merges)** of the audit workflow.
