# Reality — Workflows

End-to-end traces of how data and control actually flow through the system as it stands. One file per workflow (e.g. `market-data-ingest.md`, `order-submission.md`, `daily-snapshot.md`). Each file walks the path from trigger → modules touched → side effects → outputs, citing concrete code references (`path/to/file.py:NN`). Documents the **as-is** behaviour, including known quirks.

Created as part of the Phase 0 audit-workflow scaffolding. Files here will be produced in **Phase 1, Track 1 (reality docs)** of the audit workflow.
