# Reality — Functionality

Plain-English descriptions of what the system **actually does today**, feature by feature, derived from reading the code (not the product docs or roadmap). One file per top-level functional area; filenames should be lowercase-kebab-case (e.g. `portfolio-valuation.md`, `ibkr-sync.md`). Each file should describe observable behaviour, inputs, outputs, and current limitations — not aspirations.

Created as part of the Phase 0 audit-workflow scaffolding. Files here will be produced in **Phase 1, Track 1 (reality docs)** of the audit workflow.
