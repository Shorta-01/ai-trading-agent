# Gap Analysis

Where reality diverges from intent. Each file pairs the as-is behaviour (from `docs/reality/`) against the intended behaviour (from `docs/intent/` and existing product docs) and lists the deltas. Expect a top-level `gaps.md` plus per-area files (e.g. `gaps-valuation.md`, `gaps-ibkr-sync.md`). Each gap entry should state: what's intended, what exists, the size of the delta, and why it matters.

Created as part of the Phase 0 audit-workflow scaffolding. Files here will be produced in **Phase 1, Track 3 (gap analysis)** of the audit workflow.
