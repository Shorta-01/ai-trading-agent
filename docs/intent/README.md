# Intent

Statements of intended behaviour: what the system **should** do, independent of what it currently does. Sourced and consolidated from existing product docs (`docs/product/`, `docs/product-scope.md`, `docs/roadmap.md`, etc.) and from new decisions made during the audit. Used as the "intended" side of gap analysis. One file per area; keep prescriptive, not descriptive.

Created as part of the Phase 0 audit-workflow scaffolding. Files here will be drafted during **Phase 1 (to anchor gap analysis)** and refined through **Phase 3 (improvement memo)** of the audit workflow.
